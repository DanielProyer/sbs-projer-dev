### Berechnungslogik

```
Pikett-Betrag = Basis (80 CHF) + (Anzahl_Feiertage × 80 CHF)

Beispiele:
- Normale Woche: 80 CHF
- Woche mit 1 Feiertag: 80 + 80 = 160 CHF
- Woche mit 2 Feiertagen: 80 + 160 = 240 CHF (Weihnachten/Neujahr)
```

**Beispiele aus Monatsrechnungen:**
```
03.11.2025  KW 45  Pikett  Feiertage: 0   160.00 CHF (exkl. MWST = 148 CHF?)
22.12.2025  KW 52  Pikett  Feiertage: 2   320.00 CHF (exkl. MWST = 296 CHF?)
```

⚠️ **Diskrepanz entdeckt!**
- Preisliste sagt: 80 CHF/Tag + 80 CHF/Feiertag
- Monatsrechnungen zeigen: 160 CHF (KW 45) und 320 CHF (KW 52)

**Mögliche Erklärungen:**
1. Pikett gilt für 2 Tage (Wochenende): 80 × 2 = 160 CHF
2. Oder: Preis wurde von Heineken erhöht (80 → 160 CHF)
3. Oder: Preisliste ist veraltet

**→ KLÄRUNGSBEDARF: Stimmt die Preisliste noch?**

```
80 CHF × 2 Tage = 160 CHF (exkl.)

```

## Integration in die App: Preisberechnung

### Datenmodell: Preis-Tabelle

```sql
Preise {
  ID: INT (Primary Key)
  Kategorie: ENUM('Service', 'Stoerung', 'Eroeffnung', 'Pikett', 'Eigenauftrag', 'Montage')
  Sub_Kategorie: VARCHAR(50)  -- z.B. "Grundtarif eigen", "Störung 3"
  Preis_Exkl_MWST: DECIMAL(10,2)
  Preis_Inkl_MWST: DECIMAL(10,2)
  Bedingung: TEXT
  Gueltig_Ab: DATE
  Gueltig_Bis: DATE
  Erstellt_Am: DATETIME
  Aktualisiert_Am: DATETIME
}
```

### Preislisten-Verwaltung in der App

**Wichtig**: Heineken kann Preise ändern!

**App-Funktionen:**
1. **Preisversionierung** - Alte Preise für historische Daten bewahren
2. **Preis-Import** - Neue Preisliste von Heineken importieren (Excel-Upload)
3. **Automatische Anpassung** - Alle zukünftigen Aufträge mit neuen Preisen berechnen
4. **Historische Integrität** - Alte Aufträge behalten ihre ursprünglichen Preise

**Workflow Preis-Update:**
```
1. Heineken sendet neue Preisliste (Excel)
2. App-Admin lädt Excel hoch
3. App parst Excel und zeigt Änderungen:
   "Preisänderungen:
    - Grundtarif eigen: 69 → 72 CHF (+4.3%)
    - Störung 3: 90 → 95 CHF (+5.6%)
    - ..."
4. Admin bestätigt
5. Neue Preise ab Datum X gültig
6. Alte Aufträge behalten alte Preise
7. Neue Aufträge verwenden neue Preise
```

### Automatische Preisberechnung (Pseudocode)

```python
def berechne_service_preis(anlage, datum):
    """
    Berechnet Service-Preis basierend auf Anlage und aktuellen Preisen
    """
    # Hole aktuelle Preise für Datum
    preise = get_preise(datum)

    # Basis: Grundtarif
    if anlage.nur_heineken:
        basis = preise['Grundtarif eigen']
    elif anlage.orion:
        basis = preise['Grundtarif Orion']
    else:
        basis = preise['Grundtarif fremd']

    # Zusatzleitungen
    zusatz = 0
    for leitung in anlage.leitungen:
        if leitung.typ == 'eigen':
            zusatz += preise['Zusatzleitung eigen']
        else:
            zusatz += preise['Zusatzleitung fremd']

    # Bergkunde
    if anlage.betrieb.bergkunde:
        zusatz += preise['Anfahrtspauschale Bergkunden']

    # Zwischensumme
    total_exkl = basis + zusatz

    # MWST
    mwst = total_exkl * 0.081
    total_inkl = total_exkl + mwst

    return {
        'basis': basis,
        'zusatz': zusatz,
        'total_exkl_mwst': total_exkl,
        'mwst': mwst,
        'total_inkl_mwst': total_inkl
    }
```

```python
def berechne_stoerung_preis(bereich, betrieb, anfahrt_km, wochenende, datum):
    """
    Berechnet Störungs-Preis basierend auf Bereich und Faktoren
    """
    preise = get_preise(datum)

    # Basis-Preis nach Bereich
    if betrieb.bergkunde:
        basis = preise[f'Störung {bereich} Bergkunde']
    else:
        basis = preise[f'Störung {bereich} Normal']

    # Anfahrtspauschale (nur für Normalkunden)
    zusatz = 0
    if not betrieb.bergkunde:
        if anfahrt_km < 80:
            zusatz += preise['Anfahrtspauschale <80km']
        else:
            zusatz += anfahrt_km * preise['Anfahrtspauschale >80km']

    # Wochenend-Zuschlag
    if wochenende:
        zusatz += preise['Wochenend-Zuschlag']

    # Zwischensumme
    total_exkl = basis + zusatz

    # MWST
    mwst = total_exkl * 0.081
    total_inkl = total_exkl + mwst

    return {
        'basis': basis,
        'zusatz': zusatz,
        'total_exkl_mwst': total_exkl,
        'mwst': mwst,
        'total_inkl_mwst': total_inkl
    }
```

### 1. Preise sind variabel!
- Heineken kann Preise jederzeit anpassen
- App muss Preisversionierung unterstützen
- Historische Aufträge mit alten Preisen beibehalten

### 2. Komplexe Berechnungslogik
- Basis-Preis + Zusatzpositionen
- Bergkunden vs. Normalkunden
- Anfahrtspauschalen nach km
- Wochenend-Zuschläge
- MWST immer 8.1%

### 3. Transparenz erforderlich
- Kunden müssen Preisaufschlüsselung sehen
- "Warum kostet mein Service 150 CHF?"
  → Grundtarif 69 + Zusatzleitung 18 + Bergpauschale 180 + MWST = 288 CHF

### 4. Automatisierung möglich
- Alle Preise können automatisch berechnet werden
- Keine manuelle Eingabe mehr nötig
